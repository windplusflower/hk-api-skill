# HK API Skill 检查清单

## 索引结构检查

- [ ] 分类清晰：用户有需求时能快速找到对应文档
- [ ] 分类无重叠：每个文档有明确的归属
- [ ] 导航完整：没有文档被遗漏在索引之外

## 内容重复检查

- [ ] 无重复内容：同样的内容不在多个文档中重复出现
- [ ] 交叉引用：相关内容使用链接而非复制

## 通用范式检查

**原则**：Skill 应该是通用范式，可复用的知识

### 应该包含

- ✅ **类别 1**: 如何使用和修改游戏内已有的组件
  - 示例：HeroController、HealthManager、PlayMakerFSM 等游戏原生类
  
- ✅ **类别 2**: 如何创建和引入全新的自定义组件
  - 示例：自定义 MonoBehaviour、新 FSM、新资源等

### 不应该包含

- ❌ **特定 Mod 的自定义组件**：无法复用的内容
  - 示例：某个 Mod 特有的 BossController、CustomSpell 等
  
- ❌ **特定 Mod 的业务逻辑**：与具体 Mod 强耦合的内容

## 何时使用此清单

- 新增 skill 文档时
- 重构 skill 结构时
- 用户反馈找不到内容时
- 定期维护检查（建议每月一次）
